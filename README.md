# Wallet Service

REST API сервис для управления кошельками с поддержкой конкурентных операций.  
Приложение реализовано на Java 17 + Spring Boot 3.

---

## Описание

Сервис предоставляет API для:

▪ Пополнения кошелька (DEPOSIT)  
▪ Списания средств (WITHDRAW)  
▪ Получения текущего баланса кошелька  

Особое внимание уделено корректной работе в конкурентной среде:

▪ поддержка высокой нагрузки (1000 RPS на один кошелек)  
▪ защита от race condition  
▪ отсутствие потери обновлений при параллельных запросах  
▪ транзакционная обработка операций

## Конкурентная обработка

Для предотвращения race condition используется пессимистическая блокировка. Это гарантирует:

▪ корректное изменение баланса при параллельных запросах
▪ отсутствие потери данных
▪ последовательную обработку операций по одному кошельку

Кроме того, все операции изменения баланса выполняются внутри транзакций.

---

## API

### POST /api/v1/wallet

Изменение баланса кошелька.

#### Request body

```json
{
  "walletId": "11111111-1111-1111-1111-111111111111",
  "operationType": "DEPOSIT",
  "amount": 1000
}
```

#### Параметры

| Поле            | Описание                  |
|-----------------|---------------------------|
| `walletId`      | UUID кошелька             |
| `operationType` | `DEPOSIT` или `WITHDRAW`  |
| `amount`        | сумма операции            |

#### Возможные ответы

| Статус            | Описание               |
|-------------------|------------------------|
| `200 OK`          | операция выполнена     |
| `404 Not Found`   | кошелек не найден      |
| `409 Conflict`    | недостаточно средств   |
| `400 Bad Request` | невалидный JSON        |

---

### GET /api/v1/wallets/{walletId}

Получение текущего баланса кошелька.

#### Пример

```
GET /api/v1/wallets/11111111-1111-1111-1111-111111111111
```

#### Response

```
1000
```

## Запуск проекта

1. Клонировать репозиторий

```bash
git clone https://github.com/USERNAME/wallet-service.git
cd wallet-service
```

2. Запустить приложение

```bash
docker-compose up --build
```

После запуска приложение будет доступно по адресу:

```
http://localhost:8080
```
